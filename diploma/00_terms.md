# ТЕРМИНЫ И ОПРЕДЕЛЕНИЯ

**Виртуальная лаборатория** — интерактивная образовательная среда, предоставляющая обучающемуся возможность выполнять практические задания в симулированном или программном окружении без необходимости физического доступа к оборудованию или серверной инфраструктуре.

**WebAssembly (WASM)** — бинарный формат инструкций и соответствующая стековая виртуальная машина, стандартизированная консорциумом W3C, обеспечивающая выполнение кода в браузере с производительностью, близкой к нативной, и в изолированной среде выполнения.

**Progressive Web Application (PWA)** — веб-приложение, соответствующее набору технических требований (манифест, Service Worker, HTTPS), позволяющих установить его на устройство пользователя и работать в режиме офлайн подобно нативному приложению.

**Local-First** — архитектурный паттерн, при котором локальное хранилище на устройстве пользователя является первичным источником истины; все операции выполняются локально и без задержки, а сеть используется исключительно для фоновой синхронизации состояния между участниками.

**CRDT (Conflict-free Replicated Data Type)** — семейство структур данных, математически гарантирующих достижение единого согласованного состояния на всех репликах при асинхронном применении операций в произвольном порядке, без необходимости в центральном координаторе. Свойства: коммутативность, ассоциативность, идемпотентность.

**Service Worker** — скрипт, выполняемый браузером в фоновом потоке независимо от страницы; перехватывает сетевые запросы и реализует стратегии кэширования, обеспечивая офлайн-доступность ресурсов приложения.

**IndexedDB** — встроенная в браузер объектно-ориентированная база данных, доступная через асинхронный API; используется для хранения структурированных данных (курсы, лабораторные работы, сессии) на стороне клиента без ограничений размера, характерных для localStorage.

**Web Worker** — механизм браузера, позволяющий выполнять JavaScript-код в отдельном потоке, изолированном от основного потока пользовательского интерфейса; используется для запуска ресурсоёмких операций (компиляция и выполнение WASM-кода) без блокировки UI.

**Sandbox (изолированная среда, песочница)** — механизм ограничения доступа исполняемого кода к системным ресурсам; в контексте данной работы реализован через Web Worker с принудительным завершением (`worker.terminate()`) по истечении таймаута.

**SignalR** — библиотека реального времени для ASP.NET Core, абстрагирующая транспортный уровень (WebSocket, Server-Sent Events, Long Polling) и предоставляющая RPC-интерфейс между клиентом и сервером; используется в качестве ретранслятора CRDT-дельт между участниками сессии.

**Совместное редактирование (Collaborative editing)** — режим работы, при котором несколько пользователей одновременно вносят изменения в один документ, при этом все изменения немедленно видны остальным участникам без конфликтов и потери данных.

**WASM-движок** — реализация интерпретатора или компилятора определённого языка программирования, скомпилированная в формат WebAssembly и выполняемая в браузере; в данной работе используются Pyodide (Python), Wasmoon (Lua 5.4), sql.js (SQLite), нативный JS-движок V8 для JavaScript.

**Бесконфликтная репликация** — свойство распределённой системы, при котором операции от разных узлов могут применяться в любом порядке и дают идентичный результат, исключая необходимость разрешения конфликтов вручную.

**Курс** — структурированная учебная единица, содержащая упорядоченный набор лабораторных работ, объединённых общей тематикой и последовательностью освоения.

**Лабораторная работа** — учебное задание в рамках курса, включающее описание задачи, исходные файлы (шаблоны кода), допустимые к редактированию и защищённые от изменения файлы, а также набор автоматически проверяемых тест-кейсов.

**Тест-кейс** — пара «входные данные (stdin) — ожидаемый вывод (stdout)», используемая для автоматической проверки корректности кода студента методом сравнения фактического вывода программы с эталонным.

**Monaco Editor** — редактор кода с открытым исходным кодом, лежащий в основе Visual Studio Code; предоставляет подсветку синтаксиса, автодополнение, форматирование и полноценный API для программной интеграции, в том числе множественные модели документов (ITextModel).

**Presence (присутствие)** — механизм отображения актуального состояния участников сессии в реальном времени: список подключённых пользователей с именами и цветовой идентификацией, реализованный через `Y.Map('presence')` в рамках общего Y.Doc.

**Edge Computing** — парадигма вычислений, при которой обработка данных выполняется на устройствах конечных пользователей (на «краю» сети), а не на централизованных серверах; снижает задержку и нагрузку на инфраструктуру.

**RTT (Round-Trip Time)** — время прохождения сетевого запроса от клиента до сервера и обратно; является ключевой характеристикой, определяющей воспринимаемую задержку при серверном исполнении кода.
