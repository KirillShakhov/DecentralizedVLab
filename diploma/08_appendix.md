### Б.1 Диаграмма вариантов использования (Use-Case)

```
┌──────────────────────────────────────────────────────────────┐
│                     Виртуальная лаборатория                   │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                                                     │    │
│  │   (Создать курс)────────────────────────────┐      │    │
│  │   (Редактировать лабораторную)              │      │    │
│  │   (Добавить тест-кейсы)                     │      │    │
│  │   (Экспортировать курс в JSON)              │      │    │
│  │                                             │      │    │
│  │   (Импортировать курс из JSON)──────────────┤      │    │
│  │   (Открыть лабораторную)                    │      │    │
│  │   (Редактировать код)                       │      │    │
│  │   (Запустить код)                           │      │    │
│  │   (Запустить тесты)─────────────────────────┤      │    │
│  │   (Совместно редактировать код)             │      │    │
│  │   (Просматривать участников сессии)         │      │    │
│  │   (Скопировать ссылку на сессию)            │      │    │
│  │   (Сбросить код к шаблону)                  │      │    │
│  │   (Запустить профилировщик)                 │      │    │
│  │   (Скачать движок офлайн)                   │      │    │
│  │                                             │      │    │
│  └─────────────────────────────────────────────┘      │    │
│                                                        │    │
└──────────────────────────────────────────────────────────────┘
         ▲                        ▲
   ┌─────┴──────┐          ┌──────┴──────┐
   │ Преподаватель│          │   Студент   │
   └─────────────┘          └─────────────┘
```

**Акторы:**
- **Преподаватель** — создаёт курсы, добавляет лабораторные работы и тест-кейсы, экспортирует/импортирует курсы, настраивает read-only файлы.
- **Студент** — открывает и выполняет лабораторные работы, редактирует код, запускает тесты и профилировщик, участвует в совместных сессиях.

**Примечание:** Роли не требуют аутентификации в текущей реализации — профиль пользователя создаётся локально при первом запуске.

### В.3 Хранение CRDT-снапшотов (localStorage)

CRDT-снапшоты Y.Doc не хранятся в IndexedDB, а сохраняются в localStorage по соображениям синхронности API:

```
localStorage
├── vlab_ydoc_{roomId}     string  (base64(Y.encodeStateAsUpdate(ydoc)))
├── vlab_selected_compiler string  ("python" | "lua" | ...)
└── vlab_user              string  (JSON: { id, username, color })
```

Ключ `vlab_ydoc_{roomId}` обновляется при каждом изменении Y.Doc через обработчик события `'update'`. При следующем открытии той же комнаты состояние восстанавливается через `Y.applyUpdate`.

## Приложение Д. Скриншоты реализованного приложения

*Скриншоты выполненного приложения прилагаются в отдельных файлах изображений. Ниже приводятся описания каждого экрана.*

**Д.1 Главная страница (HomePage)**
Отображает сетку курсов. В заголовке — кнопка импорта JSON-файла курса. При отсутствии курсов показывается пустое состояние с предложением импортировать первый курс.

**Д.2 Редактор курса (CourseEditorPage)**
Список лабораторных работ в виде раскрывающихся панелей (Accordion). Каждая лаборатория содержит: поля названия и описания, список файлов с редактором содержимого и toggleом read-only, список тест-кейсов с полями stdin и ожидаемого вывода. Кнопки «Экспортировать» и «Сохранить» в заголовке.

**Д.3 Рабочая область (Workspace)**
Три колонки: FileTree (200px) | Monaco Editor (flex) | Terminal + TestPanel (320px). Верхняя панель: название лабораторной, код комнаты, выбор языка, иконки управления, список участников, статус движка.

**Д.4 Многофайловый редактор**
Вкладки открытых файлов с иконками по расширению (🐍 .py, 🟨 .js, 🌙 .lua, 🗄️ .sql). Активная вкладка подчёркнута синей линией. Кнопка закрытия появляется при наведении.

**Д.5 Панель тестирования (TestPanel)**
Список тест-кейсов со статусами: ⏳ pending → 🔄 running → ✅ pass / ❌ fail. Прогресс-бар и счётчик пройденных тестов. При раскрытии провалившегося теста — сравнение фактического и ожидаемого вывода.

**Д.6 Профилировщик (ProfilerPanel)**
Модальное диалоговое окно. Настройка числа итераций, кнопка запуска. После завершения: 20-bin гистограмма распределения, таблица статистик (min/avg/median/p95/σ), сравнительная строка для серверной оценки, кнопка «Экспортировать отчёт».

**Д.7 Список участников (ParticipantList)**
Цветные аватары-буллиты с именами участников текущей сессии в верхней панели Workspace. Обновляются в реальном времени через Y.Map('presence').

**Д.8 Страница настроек (SettingsPage)**
Информация о кэшированных WASM-движках (Pyodide, Fengari, sql.js) с размером и датой кэширования. Кнопки очистки кэша для каждого движка и всего Cache Storage.
