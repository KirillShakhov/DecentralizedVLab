# Структура ВКР

**Тема:** Исследование технологий и инструментов разработки виртуальных лабораторий для технических предметных областей

---

## Содержание

### Термины и определения (3–4 стр.)
Виртуальная лаборатория, WebAssembly, PWA, Local-First, CRDT, Service Worker, IndexedDB, Web Worker, Sandbox, SignalR, Совместное редактирование, WASM-движок, Бесконфликтная репликация, Курс/Лабораторная/Тест-кейс, Monaco Editor, Presence.

### Введение (4–5 стр.)
- Актуальность: серверные Docker-платформы дороги и медленны
- Объект: платформа виртуальных лабораторий с клиентским исполнением кода
- Предмет: технологии WebAssembly, CRDT, PWA
- Цель: снижение затрат и повышение скорости отклика (Local-First)
- Задачи: анализ → архитектура → WASM-песочница → CRDT → эксперимент
- Научная значимость: синтез CRDT + WASM для образования
- Практическая значимость: -90% затрат на инфраструктуру вуза

### Глава 1. Исследование предметной области (25–30 стр.)

**1.1 Описание предметной области**
- Виртуальные лаборатории в образовании
- Проблемы серверного подхода: RTT, стоимость, зависимость от сети
- Тренд на Edge Computing и Local-First

**1.2 Обзор и сравнение существующих решений**

Сравнительная таблица платформ (Stepik, Replit, JupyterHub, CodeSandbox, PhET, Pyodide):
- критерии: исполнение кода, офлайн, совместная работа, стоимость

Сравнение технологий исполнения в браузере:
- JS-интерпретаторы (Skulpt) vs WebAssembly (Pyodide, Fengari, sql.js)

Сравнение подходов синхронизации:
- OT (требует арбитра, не Local-First) vs CRDT (коммутативность, ассоциативность, идемпотентность)

**1.3 Определение требований**
- Функциональные: Python/JS/Lua/SQLite, многофайловость, курсы+тесты, совместная работа, офлайн, импорт/экспорт JSON, профилировщик
- Нефункциональные: отклик <50 мс, таймаут 5 сек, PWA

### Глава 4. Экспериментальная оценка (15–20 стр.)

**4.1 Сравнительный анализ производительности**
- Методология: N=100 итераций, эталон FizzBuzz N=100 (Python)
- Оценка сервера: RTT + Docker warm 150 мс / cold 450 мс
- Результат: WASM ~7 мс среднее vs сервер ~185 мс → **ускорение ~26x**
- Гистограмма распределения

**4.2 Экономическое обоснование**
- 1000 пользователей: сервер ~45 000 руб./мес. vs WASM ~4 500 руб./мес.
- **Снижение затрат ~90%**
- График масштабирования затрат
